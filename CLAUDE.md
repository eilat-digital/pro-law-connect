# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Hebrew-language (RTL) landing site for the law office "שירן שושני – חברת עורכי דין, נוטריון וגישור" (family law, notary, mediation, based in Eilat). Vite + React 18 + TypeScript + Tailwind + shadcn/ui, originally generated by Lovable (see `.lovable/plan.md`). All user-facing text must be in Hebrew; the whole document is RTL (`direction: rtl` set globally in `src/index.css`).

## Commands

```sh
npm run dev        # dev server on port 8080 (vite.config.ts)
npm run build      # production build
npm run lint       # eslint
npx tsc --noEmit -p tsconfig.app.json   # type-check (no test suite exists)
```

## Architecture

SPA with react-router. Routes are declared in `src/App.tsx`:

- `/` — `src/pages/Index.tsx` (the entire landing page: hero, quotes, מתגשרים banner, 20 practice areas, about, services, testimonials, articles, contact strips)
- `/mitgashrim` — mediation program "מתגשרים" + the nonprofit "אור עליון – נהורא עלאה" (section `#amuta`)
- `/accessibility` — accessibility statement (legally required; linked from the accessibility widget and footer)
- `/privacy` — privacy policy per Israeli Privacy Protection Law Amendment 13 (legally required)

### Where content lives (edit here, not inline)

- `src/content/homepage.json` — CMS-managed content: hero title/description, about paragraphs, the 20 practice areas (title+desc), and contact details (phone, mobile, fax, email, address). Edited by the site owner through Decap CMS at `/admin/` (config: `public/admin/config.yml`, backend: git-gateway → commits to `main`). Field names in `config.yml` must stay in sync with this JSON's shape.
- `src/config/site.ts` — imports contact details from `homepage.json`, derives intl phone formats and the WhatsApp link, and holds social URLs and site name/tagline. Every component imports from here; never hardcode contact info elsewhere.
- `src/pages/Index.tsx` — practice-area icons are mapped by exact Hebrew title in `iconByTitle` (fallback: Scale); remaining data arrays: `services`, `testimonials` (placeholder text awaiting real client quotes), `articles`.
- The authoritative content brief from the client is `../תוכן אתר.docx` (one directory above the repo, outside git).

### Decap CMS

`public/admin/` (index.html + config.yml) is served statically. It only works on the deployed Netlify site after enabling Identity + Git Gateway; there is no local CMS backend. CMS saves commit directly to `main`, which triggers a Netlify rebuild.

### Page shell pattern

Every page renders `Header` + `Footer` + `FloatingButtons` (WhatsApp/phone) + `AccessibilityWidget` itself — there is no shared layout component. New pages must include all four and a `Helmet` block (react-helmet-async) with Hebrew title/description.

### Header navigation

`Header.tsx` nav items scroll to section ids on `/`; from other pages it navigates to `/#<id>`, and `Index.tsx` handles the hash-scroll in a `useEffect`. Keep the nav order: תחומי עיסוק before אודות (explicit client requirement).

### Styling system

- Colors are HSL CSS variables in `src/index.css` (`--background`, `--accent` = gold, etc.) consumed via Tailwind tokens; the "legacy" tokens (`--gold`, `--navy`, `--cream`) are remapped to the same palette.
- Accessibility widget (`src/components/AccessibilityWidget.tsx`) works by setting `data-a11y-*` attributes on `<html>`; the corresponding CSS rules live in `src/index.css`. Any new styling must not break under `data-a11y-contrast="high"` or `data-a11y-font` scaling.
- `.font-tanakh` (Frank Ruhl Libre) is the biblical-style font used for the client's personal quotes (`QuoteStrip` in Index.tsx).
- CSS `@import` statements must stay above the `@tailwind` directives in `index.css` (vite:css warns otherwise).

### Language switcher

`src/components/LanguageSwitcher.tsx` uses the Google Translate cookie mechanism (`googtrans` cookie + injected script), not real i18n. Source language is `he`. Related CSS (banner hiding) is at the bottom of `index.css`. Elements that must never be machine-translated get the `notranslate` class.

### Images

Static images are plain files in `public/` (`/logo.jpeg`, `/shoshani-portrait.jpeg`, `/mitgashrim-logo.jpeg`). The `src/assets/*.asset.json` files are the old Lovable-hosted asset pointers — they only resolve on Lovable hosting; don't add new ones.

### Contact form (Netlify Forms)

`src/components/sections/ContactForm.tsx` posts urlencoded data to `/` with `form-name=contact` (Netlify Forms). A hidden static mirror of the form lives in `index.html` so Netlify's build-time scanner registers it — keep the field names (`name`, `phone`, `email`, `message`, `bot-field`) in sync between both files. Submissions only actually deliver when the site is hosted on Netlify.

### SEO invariants

Exactly one `<h1>` per page; every `<img>` must have a Hebrew `alt`. Static fallback title/description live in `index.html`; per-page values come from each page's `Helmet` block.
